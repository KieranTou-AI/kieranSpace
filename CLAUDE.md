# CLAUDE.md

## Project Overview

kieranSpace — personal blog & tutorial site built with Next.js. No backend, no database. Content lives as MDX files in the repo.

## Tech Stack

- **Next.js** (App Router) — static site generation, no SSR/server needed
- **Tailwind CSS** — styling
- **shadcn/ui** — UI components (imported from `@/components/ui`)
- **MDX** — blog content, stored in `content/`
- **Cloudflare R2** — image hosting (use R2 URLs for images in MDX, don't store images in the repo)
- **Vercel** — deployment (auto-deploys on push to `main`)

## Project Structure

```
kieranSpace/
├── app/                    # Next.js App Router pages & layouts
│   ├── layout.tsx          # Root layout (Tailwind, fonts, metadata)
│   ├── page.tsx            # Home page
│   └── blog/
│       └── [slug]/         # Dynamic blog post routes
├── components/             # Reusable components
│   └── ui/                 # shadcn/ui components (generated by `npx shadcn@latest add`)
├── content/                # MDX blog posts — one file = one post
├── lib/                    # Utilities (MDX processing, etc.)
├── public/                 # Static assets (favicon, etc.)
└── package.json
```

## Conventions

### General

- Use TypeScript everywhere (`.tsx`, `.ts`)
- Prefer server components; use `"use client"` only when interactive
- Use Tailwind classes directly in JSX — no CSS modules
- Naming: kebab-case for files, PascalCase for components, camelCase for utilities

### MDX Blog Posts

- Place in `content/` with descriptive filenames: `react-hooks-guide.mdx`
- Frontmatter must include: `title`, `description`, `date` (YYYY-MM-DD), `tags: [a, b, c]`
- Images: upload to Cloudflare R2 first, then use the R2 URL in MDX. Do NOT store images in the repo.
- MDX supports embedding React components — use for interactive demos, code sandboxes, etc.

### shadcn/ui

- Add components with: `npx shadcn@latest add <component-name>`
- Components go to `@/components/ui/` — don't edit them directly; customize via Tailwind classes on the parent

## Commands

```bash
npm install              # Install dependencies
npm run dev              # Start dev server
npm run build            # Production build
npm run build && npm start  # Verify production build locally
npx shadcn@latest add <name>  # Add a shadcn component
```

## When Building This Project

1. Set up Next.js with App Router and TypeScript
2. Install Tailwind + shadcn/ui first
3. Set up the content layer (MDX processing) before building UI
4. Create the blog route pattern (`app/blog/[slug]/`)
5. Build components top-down: layout → homepage → blog list → blog detail
