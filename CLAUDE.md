## Project Overview

kieranSpace вЂ” personal blog & tutorial site built with Next.js. No backend, no database. Content lives as MDX files in the repo.

## Tech Stack

- **Next.js** (App Router) вЂ” static site generation, no SSR/server needed
- **Tailwind CSS** вЂ” styling
- **shadcn/ui** вЂ” UI components (imported from `@/components/ui`)
- **MDX** вЂ” blog content, stored in `content/`
- **Cloudflare R2** вЂ” image hosting (use R2 URLs for images in MDX, don't store images in the repo)
- **Vercel** вЂ” deployment (auto-deploys on push to `main`)

## Conventions

### General

- Use TypeScript everywhere (`.tsx`, `.ts`)
- Prefer server components; use `"use client"` only when interactive
- Use Tailwind classes directly in JSX вЂ” no CSS modules
- Naming: kebab-case for files, PascalCase for components, camelCase for utilities

### MDX Blog Posts

- Place in `content/` with descriptive filenames: `react-hooks-guide.mdx`
- Frontmatter must include: `title`, `description`, `date` (YYYY-MM-DD), `tags: [a, b, c]`
- Images: upload to Cloudflare R2 first, then use the R2 URL in MDX. Do NOT store images in the repo.
- MDX supports embedding React components вЂ” use for interactive demos, code sandboxes, etc.

### shadcn/ui

- Add components with: `npx shadcn@latest add <component-name>`
- Components go to `@/components/ui/` вЂ” don't edit them directly; customize via Tailwind classes on the parent
